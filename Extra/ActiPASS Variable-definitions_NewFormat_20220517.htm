varNames Definition Comments
SubjectID ID found using accelerometer filenames automatically  or separate key file connecting IDs to accelerometer file names
QC Auto detected preliminary quality check (QC) flag. QC flag is auto detected using predefined outlier criteria. Additionally technical errors can also generate QC flag. 
Batch A string containing ActiPASS batch process date and time
NumDays Number of days present in the accelerometer data after automatically trimming Non-wear (NW) days in the beginning and end
NumValidDays Number of valid days (default valid day criteria is based on at least 20h of wear time)
NumWorkDays Number of work days detected. A work day is a day where some time of the day is flagged as "Work" by  diary. Not available at the moment for SCAPIS. Only available when work/leisure diary is given
NumLeisureDays Number of leisure days detected. A leisure day is where some or whole part of the day is flagged as leisure, but not as work. Not available at the moment for SCAPIS. Only available when work/leisure diary is given
  Following variables are derived daily. The suffix (in this case "_1") indicate the day number. A day is a calendar day starting from 00:00:00 and ending with 23:59:59. The first and last day could be partial days. Only days determined to be valid (see above) are listed in the table.  
Date_1 The date in ISO format (yyyy-mm-dd)
Day_1 The day name in English
Weekend_1 A T/F flag whether the given day is Saturday or Sunday
DayType_1 Whether the  is a work-day or a leisure day. See definitions of number of work and leisure days above. Possible values "Work", "Leisure" or empty
DayStart_1 The start of the day in 24h format (HH:MM:SS). This is normally 00:00:00 for full days
DayStop_1 The end of the day in 24h format (HH:MM:SS). This is normally 23:59:59 for full days
  A day can be further sub-divided into domains such as 'Work' and 'Leisure' and following variables can be generated for each such sub-domain within a calendar day.  Such variables will be prefixed with the domain (Ex. Work_Duration_1) This additional domain specific data is available only when the additional diary containing such data is included in ActiPASS processing, and also the relevant domain names are specified under "Stat Domains" options before stats generation.  
     
  Variables related to exclusions, non-wear and validity. All times are given in minutes. These variables satisfy the equations:
Duration = Excluded + NonWear + ValidDuration
and
NonWear = AwakeNW +  SlpIntNW
  
Excluded_1 Time excluded from analysis by the special diary event "X"
Duration_1 Total measured time for the day (1440 minutes for full days)
NonWear_1 Time of auto or manually defined non-wear (NW) during the day
AwakeNW_1 The time of NW outside any periods flagged as sleep-intervals. There could be more than one sleep-intervals happening for a given calendar day
SlpIntNW_1 The time of NW inside any periods flagged as sleep-intervals. There could be more than one sleep-intervals happening for a given calendar day
ValidDuration_1 The valid time which can be used for analysis for this day
  Variables related to the bedtime, sleep-interval and awake time. All times are given in minutes. These variables satisfy the equation: ValidDuration = Awake + SleepInterval  
Awake_1 The time within ValidDuration of this calendar day which is not flagged as belonging to a sleep-interval
SleepInterval_1 The  time within ValidDuration of this calendar day which is flagged as belonging to a sleep-interval.
 A sleep interval is theoretically defined as the time between the first sleep onset and the last awakening. Therefore, a calendar day usually contains more than one (full or partial) sleep-intervals. This variable is calculated by counting the number of epochs (1s) which are flagged as belonging to a sleep-interval
Sleep_1 Total sleep time within ValidDuration of this calendar day. (both sleep inside and outside bedtime. For SCAPIS only sleep inside bedtime is found)
SleepInBed_1 The total times of sleep within a bedtime. This is also the time of sleep inside sleep_intervals by definition of sleep-interval.
SleepOutBed_1 The total time of sleep outside a bedtime
  Activity/Posture times and cadence related variables. These variables are defined within "Awake". All times are given in minutes. These variables satisfy the equation:                        Awake=Lie+Sit+Stand+Move+Walk+Run+Stair+Cycle+Other  
Lie_1 Time of lying. If SleepOutBed is defined, time of 'Lie' also contains that time.
Sit_1 Time of sitting
SitLie_1 Time of sitting or lying.  If SleepOutBed is defined, time of 'SitLie' also contains that time.
Stand_1 Time of standing
Move_1 The time of 'move'. 'Move' is defined as periods where person is in  standing posture with certain movements above the threshold for standing posture (ex. intermittent steps), but no purposeful walking.
StandMove_1 Time of standing or moving
Walk_1 Time of walking
Walk_Slow_1 Time of walking with a cadence lower than 100
Walk_Fast_1 Time of walking with a cadence equal or higher than 100
Run_1 Time of running
Stair_1 Time of stair-walking (all stair walking like activities are counted. i.e. terrain walking etc.)
Cycle_1 Time of cycling
Other_1 Time of activities in "Other" postures/activities which has somewhat horizontal thigh position. Time of 'Other' could be high due to various reasons such as: actual activities such gym equipment, rowing/kayaking or individual calibration issues.
     
  Time spent on different energy expenditure intensity classes. See the separate appendix for how intensity classes are derived. All times are given in minutes  
SED_1 Time spent on activities classified as sedentary 
LPA_1 Time spent on activities classified as low-physical activity 
MPA_1 Time spent on activities classified as moderate physical activity 
VPA_1 Time spent on activities classified as vigorous  physical activity
MVPA_1 Time spent on activities classified as moderate or  vigorous  physical activity
     
  Cadence related variables. Cadence is only detected for walking, running and stair-walking  
NumSteps_1 Total number of steps per this day
NumStepsWalk_1 Number of steps during walking
NumStepsRun_1 Number of steps during running
